| FR-01 | Task submission and validation (`taskq submit "<command>" [--name NAME]`). Validation: non-empty; ≤1000 chars; injection-char blacklist `;` `|` `&` `$` `>` `< `` ` ``; name uniqueness vs pending/running. Violation → exit 2 + stderr, no storage write. | FR-02 | Task executor (`taskq run <id>` / `taskq run --all`). `subprocess.run(shlex.split(command), capture_output=True, text=True, timeout=TASKQ_TASK_TIMEOUT)` — `shell=True` forbidden. State machine `pending → running → done|failed|timeout` (exit 0 → done; non-0 → failed; TimeoutExpired → timeout). Records `exit_code`, `stdout_tail` (last 2000 chars), `stderr_tail` (last 2000 chars), `duration_ms`, `finished_at`. `--all`: `ThreadPoolExecutor(max_workers=TASKQ_MAX_WORKERS)`; storage writes thread-safe via shared Lock. Single-task timeout → exit 4. | Behavior (process orchestration) | subprocess without shell=True + ThreadPoolExecutor + Lock | DRAFT | Requirements Engineer | SPEC §3 FR-02; thread-safe writes per NFR-03 |
| FR-03 | Retry + circuit breaker. Retry: on `failed`/`timeout` auto-retry up to `TASKQ_RETRY_LIMIT`; nth retry waits `TASKQ_BACKOFF_BASE × 2^n` seconds (sleep injectable for tests). Breaker (global, cross-task, cross-process): consecutive final failures ≥ `TASKQ_BREAKER_THRESHOLD` → `OPEN`; OPEN period any `run` rejected immediately (exit 3 + stderr `breaker open`, no subprocess); after `TASKQ_BREAKER_COOLDOWN` seconds → `HALF_OPEN` (one trial — success → CLOSED + counter zeroed; fail → re-OPEN). Persisted in `$TASKQ_HOME/breaker.json` (atomic write). | Behavior (resilience) | exponential backoff + FSM (CLOSED/OPEN/HALF_OPEN) + atomic persist | DRAFT | Requirements Engineer | SPEC §3 FR-03; OPEN ISSUE: AC-03.1 "n-th retry" uses n as 0-indexed or 1-indexed is not explicit in SPEC — test authors must clarify with stakeholder before writing backoff fixtures |
| FR-04 | Result TTL cache. Cache miss/expired → normal execution, write to `$TASKQ_HOME/cache.json` on `done`. Cache read/write: atomic + thread-safe (coexists with FR-02). | Behavior (memoization) | sha256 key + TTL check + atomic persist | DRAFT | Requirements Engineer | SPEC §3 FR-04; replay must skip subprocess |
